== Onlinefighter

Браузерная игра по мотивам известного проекта combats.com.

Пользователю предлагается зарегистрироваться на сайте либо войти с использованием аккаунта в одной из популярных соцсетей (vk, fb, twitter). После аутентификации на выбор предоставляются пять классов персонажей: воин, паладин, маг, вор, бард. Каждый класс имеет свой уникальный набор значений характеристик (здоровье, атака, защита, магия).

Персонаж может хранить в рюкзаке и одевать предметы пяти типов: оружие, шлем, кираса, перчатки, сапоги. Каждый предмет может быть одет только одним классом персонажей. Одетые предметы увеличивают одну из характеристик персонажа.

Персонаж может сражаться с компьютерными игроками. Победа или поражение определяются случайным образом. В зависимости от исхода сражения изменяются характеристики персонажа (уменьшается здоровье, в случае победы увеличивается опыт). На выбор предоставляется пять противников. Сражение с каждым противником происходит единократно. После пяти боев предлагается поделиться ссылкой на игру в соц. сетях.

<b>Технические требования:</b>
* RoR, MySQL
* Devise, OmniAuth
* При создании персонажа выдавать в рюкзак два одинаковых предмета
* Функионал «одеть/снять предмет» реализовать с применением AJAX

<b>Заметки по запуску:</b>
* Для работы функционала, связанного с соцсетями, необходимо предоставить id и secret_key приложений. Сделать это необходимо вручную посредством создания файлов config/config\_facebook.yml, config/config\_twitter.yml, config/config\_vkontakte.yml со следующим содержимым:

    development:
      app_id: id_приложения
      secret: secret_приложения
* Заполнить БД стартовыми значениями можно с помощью seeds.rb
* Твиттер и Вконтакте при авторизации не отдают емейл-адрес пользователя, поэтому сначала требуется залогиниться из-под Фейсбука (или зарегистрироваться на сайте без использования соцсетей), затем на users/edit добавить необходимые uid'ы к своему профилю. После этого можно логиниться из-под контакта и твиттера.
